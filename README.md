# ircbot
ircbot.c by trojanman - runs an ircbot with secure msg commands

This project pairs with [irchub](https://github.com/robertclemens/irchub) — a hub server for centralized management of IRC bot networks. irchub handles Curve25519 keypair generation, encrypted config distribution, and multi-bot coordination.


### System Requirements
    * Compiler: gcc (Tested with: GCC 13.3.0)
    * Make: GNU Make (Tested with: GNU Make 4.3)
    * OpenSSL Libs: libssl-dev (Tested with: libssl-dev 3.0.13)   (IE: apt install libssl-dev on Ubuntu)
    * LibCURL Libs: libcurl-dev

### How to use:
1) Take a look at bot.h if you want to set any custom time intervals. Defaults should be good so I recommend leaving alone unless you run into something later.

2) Run `"make && make install"`
    * "make" creates the binary
    * "make install" cleans the directory of all config and compiler objects.

3) Run `"./ircbot -setup"`
    * This loads the config wizard to create your initial config file.
    * You will be prompted to set a config password during setup.

4) Start ircbot. The bot daemonizes automatically on startup. There are two ways to provide the config password:

    * **Interactive password prompt** (most secure, manual start only) — run the bot with no password file and it will prompt for the config password each time:
        ```
        ./ircbot
        Config Password:
        ```
        The password is never written to disk and leaves no trace after the process daemonizes. This is the most secure option but requires manual intervention on every start — the bot cannot be launched automatically via cron or init scripts.

    * **Password file** (optional, unattended / crontab-friendly) — run once to generate a machine-bound encrypted password file:
        ```
        ./ircbot -p
        ```
        This creates `.ircbot.pass` (mode 0600, encrypted and bound to this machine). The bot will load it automatically on every start with no interaction required, at the cost of storing the encrypted password on disk. This is required if you want to use crontab or any other automated restart mechanism.

        Example crontab line (optional):
        ```
        */5 * * * * /home/user/ircbot/ircbot 1>/dev/null 2>/dev/null
        ```

5) To send commands to your bot, you will need to hash your DEFAULT_BOT_PASS that you defined in bot.h. The hash output is timebased and lasts for no more than 120 seconds. I have provided two methods for generating this hash:
* Linux CLI: ./bot-auth.sh <DEFAULT_BOT_PASS> will output a hash you may send to your bot: 
    * /msg botnick <hash_output> help
* IRSSI Script: bot_auth.pl. You may "/script load bot_auth.pl" with bot_auth.pl in ~/.irssi/scripts/ or place in ~/.irssi/scripts/autorun/ for automatic loading upon starting irssi. 
    * /set bot_auth_password <DEFAULT_BOT_PASS>
    * /botcmd botnick help
